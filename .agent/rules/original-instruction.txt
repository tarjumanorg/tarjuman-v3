### ALUR UX TARJUMAN (Flow Lengkap)

#### 1. Halaman Depan: "The Hook" (Tanpa Login)
User membuka `tarjuman.com` di HP mereka.

*   **Header:** Logo Tarjuman + Teks kecil: *"Jasa Penerjemah Tersumpah (Sworn) Resmi untuk Beasiswa Saudi."*
*   **Hero Section:** Tidak ada tombol "Daftar". Langsung **Dropzone Besar**.
*   **Copywriting:**
    > *"Seret dokumen Anda ke sini (Ijazah, Transkrip, Akta, SKCK)."*
    > *"Format: PDF, JPG, PNG. Kami akan hitung otomatis."*
*   **Action:** User memilih file dari galeri HP atau memotret dokumen langsung.

#### 2. Inventaris & Konfigurasi (The Transformation)
Setelah file diupload, halaman tidak berpindah, tapi berubah tampilan (state change).

*   **List File:** Menampilkan thumbnail kecil dari file yang diupload.
*   **Verifikasi Halaman (PENTING):**
    *   Sistem mendeteksi jumlah halaman.
    *   User melihat: *"Ijazah.pdf - Terdeteksi 1 Halaman."*
    *   **User Action:** Jika sistem salah baca (misal PDF gabungan), user bisa klik tombol `[+]` atau `[-]` untuk mengoreksi jumlah halaman manual. (beri tahu user untuk melakukan ini)
*   **Opsi Kecepatan (Slider):**
    *   Slider dari **1 Hari (Express)** sampai **7-12 Hari (Santai)**.
    *   Harga berubah real-time saat slider digeser.
    *   *Psikologi:* "Makin cepat, makin mahal."
*   **Upsell Fisik:**
    *   Checkbox: *"Kirim Hard Copy (Cap Basah) ke rumah saya?"*
    *   Jika dicentang -> Muncul kolom alamat (Gunakan Google Places Autocomplete agar tidak capek mengetik).

#### 3. Checkout & Identitas (The Gate)
User melihat total harga: **Rp 150.000**. User klik tombol **"Lanjut Pembayaran"**.

*   **Pop-up Login (Lazy Auth):**
    > *"Ke mana kami harus kirim hasil terjemahan?"*
    >
    > **[ Tombol Besar: Masuk dengan Google ]** (Rekomendasi Utama)
    > *Link kecil: Atau gunakan email*
*   **Logika:**
    *   Jika Google: Akun langsung dibuat, data nama & email diambil dari Google.
    *   Jika Email: User input email -> Sistem kirim "Magic Link" (tanpa password).

#### 4. Pembayaran (Indonesian Style)
*   User diarahkan ke halaman pembayaran (Integrasi Midtrans/Xendit).
*   **Metode:** Prioritaskan **QRIS** (paling cepat), Virtual Account Bank, dan E-Wallet.
*   **Status:** Setelah bayar sukses -> Redirect ke **Dashboard Pesanan**.
    *   *Pesan:* *"Dokumen sedang diverifikasi admin. Notifikasi akan masuk via Email/WhatsApp (opsional)."*

### ALUR OPERASIONAL (Admin & Sistem)

#### 5. Admin Gatekeeper (Pencegahan Curang)
*   **Admin Action:** Anda download file, cek jumlah halaman.
*   **Skenario A (Aman):** Jumlah halaman sesuai. Anda klik "Terima". Status user berubah jadi "Sedang Diterjemahkan".
*   **Skenario B (Kurang Bayar):** User upload PDF 10 halaman tapi cuma bayar 1 halaman.
    *   Anda klik **"Minta Tambahan Biaya"**.
    *   Sistem kirim email otomatis ke user dengan link pembayaran selisih harga. Proyek di-pause sampai lunas.
tampilkan hasil perhitungan sistem dan jika user mengubahnay secara manual.

#### 6. Proses Penerjemahan & Draft
*   Admin menerjemahkan di MS Word.
*   **Simpan Draft:** Admin save as **PDF dengan Watermark "DRAFT / REVISI"**.
*   Admin upload draft ke sistem.

#### 7. Review Loop (Inti UX)
User mendapat email: *"Draft terjemahan Ijazah Anda sudah siap. Silakan periksa."*

*   **User Action:** Klik link di email (Login otomatis via token).
*   **Tampilan Mobile:**
    *   Layar menampilkan PDF Draft (bisa di-zoom/pinch).
    *   Di bawah PDF, ada **Chat Box/Kolom Komentar**.
    *   *Contoh User:* "Kak, nama ibu saya ejaannya 'Siti', bukan 'City'. Mohon perbaiki."
*   **Admin:** Memperbaiki di Word -> Upload Draft V2.
*   **Finalisasi:** Jika sudah benar, user klik tombol hijau besar: **[ SETUJUI DRAFT ]**.

#### 8. Hasil Akhir (Final Delivery)
Setelah user klik "Setujui":

*   **Admin Action:** Upload file final (PDF bersih dengan gambar Cap Basah & Tanda Tangan Penerjemah Tersumpah).
*   **User Dashboard:** Tombol berubah menjadi **[ Download Dokumen Resmi ]**.
*   **Hard Copy (Jika user beli):**
    *   Admin print, cap basah asli, kirim via JNE/J&T.
    *   Admin input Resi Pengiriman ke dashboard.

### HAL TEKNIS PENTING (Untuk Developer)

1.  **Pricing Model:**
    *   Gunakan **Per Halaman Sumber (Source Page)**. Jangan hitung hasil jadi. Ini mempermudah user menghitung biaya di awal.
2.  **Auth:**
    *   Wajib **Google OAuth**. Mahasiswa Indonesia rata-rata pakai Android dan akun Google mereka selalu aktif. Ini memangkas friksi registrasi hingga 0%.
3.  **Keamanan File:**
    *   Pastikan file yang diupload user tersimpan di private storage, bukan public folder. Dokumen ini sensitif (Data Pribadi).
4.  **Mobile Responsiveness:**
    *   Pastikan PDF Viewer di HP lancar (bisa pakai library seperti `react-pdf` atau `pdf.js` yang mobile friendly). Jangan paksa user download dulu baru bisa lihat draft.

### REKOMENDASI COPYWRITING (Bahasa Indonesia)

*   Jangan pakai kata kaku seperti *"Submit Order"*. Gunakan *"Terjemahkan Sekarang"*.
*   Di bagian checkout, tambahkan *Trust Badge*:
    *   *"Penerjemah Tersumpah Resmi SK Gubernur/Kemenkumham"*
    *   *"Diakui Kedutaan & Universitas Saudi"*
*   Di bagian upload, beri contoh:
    *   *"Upload dokumen beasiswa kamu di sini. Kami sudah berpengalaman menerjemahkan untuk persyaratan Study in Saudi."*

Dengan alur ini, `Tarjuman` akan terasa seperti aplikasi modern (seperti Gojek/Traveloka) namun khusus untuk jasa penerjemahan resmi.

### UX Writing Nuances (Indonesian)

*   **Avoid:** "Submit", "Register", "Checkout".
*   **Use:** "Kirim", "Mulai", "Lanjut".
*   **Currencies:** Format properly: **Rp 150.000** (not Rp150k or 150000).
*   **Tone:** Helpful, polite, official.
    *   *Error Message:* *"Waduh, koneksi terputus. Coba lagi ya."*
    *   *Success:* *"Alhamdulillah, dokumenmu sudah kami terima."*

### Desktop Adaptation (Quick Note)
On desktop, use a **Split Screen** layout for the "Review" page:
*   **Left:** Original Document (Scan).
*   **Right:** Translated Draft.
*   **Far Right Sidebar:** Comments/Chat.
This allows the student to compare line-by-line without flipping tabs.
